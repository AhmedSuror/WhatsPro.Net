# WhatsPro.Net SDK — Implementation Plan

## Goal

Build a production-ready, NuGet-publishable C# class library that wraps the Whats-Pro REST API. The SDK must be fully asynchronous, strongly typed, and completely hide HTTP/encryption details from consumers. Targets `net48` and `netstandard2.0`.

## Design Decisions (from interview)

| Decision | Choice |
|---|---|
| AES encryption | Transparent — SDK encrypts all payloads automatically |
| Encryption key | Default to `abcd123456789ABCD`, overridable on `WhatsProOptions` |
| Authentication | JWT-only (email+password → auto-login, token caching, re-auth on expiry) |
| Feature scope | All 6 areas: Auth, Dashboard, Clients, Groups, Sessions, Messages |
| CancellationToken | Optional last parameter on every async method |
| Response shape | `WhatsProResponse<T>` wrapper (Success, Message, Data) |
| Crypto library | `System.Security.Cryptography` only — manual EVP_BytesToKey |
| Pagination | `PagedResponse<T>` with explicit page params, no auto-iteration |
| CI scaffolding | Placeholder files only, not functional |
| Serialization | `System.Text.Json` only |
| HTTP | `HttpClient` only, no RestSharp/Flurl/Refit |

#### Internal HTTP Layer (`Internal/`)

##### [NEW] [WhatsProHttpClient.cs](file:///d:/AS Lab/Desktop/Libraries/WhatsPro.Net/src/WhatsPro/Internal/WhatsProHttpClient.cs)
Internal transport. Responsibilities:
- Holds the `HttpClient` instance
- Adds Bearer token to authenticated requests
- Serializes request body → JSON → AES encrypt → `{"payload": "..."}` wrapper
- Sends HTTP requests (GET, POST, PUT)
- Deserializes responses to `WhatsProResponse<T>`
- Maps HTTP errors to typed exceptions

##### [NEW] [AuthenticationManager.cs](file:///d:/AS Lab/Desktop/Libraries/WhatsPro.Net/src/WhatsPro/Internal/AuthenticationManager.cs)
- Holds cached JWT token
- Calls `/user/login` on first request
- Re-authenticates when token expires (168h TTL from API)
- Thread-safe token refresh using `SemaphoreSlim`

---

#### Serialization/

##### [NEW] [PayloadEncryptor.cs](file:///d:/AS Lab/Desktop/Libraries/WhatsPro.Net/src/WhatsPro/Serialization/PayloadEncryptor.cs)
CryptoJS-compatible AES encryption/decryption using `System.Security.Cryptography`:
- `EVP_BytesToKey` implementation (MD5-based key derivation)
- AES-256-CBC with PKCS7 padding
- Generates random 8-byte salt
- Output format: `"Salted__" + salt + ciphertext` → Base64

##### [NEW] [JsonOptions.cs](file:///d:/AS Lab/Desktop/Libraries/WhatsPro.Net/src/WhatsPro/Serialization/JsonOptions.cs)
Shared `JsonSerializerOptions` with `snake_case` naming policy.
